# 局限性与未来工作

## 当前局限性

### 1. 数据集的限制

- **规模偏小**：数据集仅包含 11,778 条样本，来自 8 位真实说话人和 7 种 AI 变声系统。现实场景中的深度伪造检测需要覆盖更广泛的声音分布和生成方式。
- **说话人同质性**：8 位真实说话人均为英语母语的公众人物（拜登、奥巴马、马斯克等）。模型对其他语言、口音或人群的泛化能力未经验证。
- **数据集固定不变**：训练数据是静态的。随着 AI 语音合成技术快速迭代（如 2024–2025 年间涌现的新 TTS/VC 模型），现有模型无法自动适应新型伪造方式。
- **音频质量单一**：样本采集条件相对干净，尚未针对背景噪声、压缩失真或电话质量音频进行评估。

### 2. 特征工程的限制

- **26 维均值特征**：每段音频仅被压缩为一个 26 维向量（对时间维度取均值），丢失了时序动态信息——而时序特征往往是区分真实与合成语音的关键。
- **缺乏韵律和音调建模**：基频（F0）轮廓、语速、节奏等韵律特征未被纳入，而这些恰恰是检测语音合成的重要线索。
- **特征集固定**：特征提取流程（`extract_features.py`）使用预设的 MFCC 和频谱统计量，不支持端到端的自动特征学习。

### 3. 模型能力的限制

- **不支持实时流式处理**：所有模型均需完整音频文件作为输入，无法对实时音频流进行检测。
- **仅支持二分类**：系统只能判断"真实"或"伪造"，无法识别具体使用了哪种 AI 合成系统，也无法量化篡改程度。
- **ML 模型的域迁移能力弱**：在平衡数据集上训练的机器学习模型，面对格式、时长或录音环境差异较大的上传音频时，性能可能明显下降。
- **SpecRNet 仅使用 CPU**：深度学习模型在 Docker 容器内以 CPU 运行，单文件推理耗时约 1–3 秒，未配置 GPU 加速。

### 4. 平台与部署的限制

- **无用户身份验证**：平台对所有能访问网络的用户开放。在共享部署场景下，多用户同时运行模型可能引发资源竞争。
- **无用户会话隔离**：模型结果以全局缓存形式存储，而非按用户隔离。不同用户同时运行同一模型可能相互干扰。
- **单节点部署**：当前 Docker Compose 方案将所有服务运行在单个容器中，不支持负载均衡或水平扩展。
- **自签名 HTTPS**：HTTPS 支持依赖自签名证书配合 Cloudflare "Full" SSL 模式，不适合无 Cloudflare 代理的生产环境（需引入 Let's Encrypt 等正规证书）。
- **上传文件大小限制**：音频上传上限为 200 MB，超长录音或无损格式文件可能超出此限制。

### 5. 教学体验的限制

- **缺乏引导式教程**：用户需自行探索，没有针对机器学习概念不熟悉的学生设计的逐步引导流程。
- **可解释性不足**：预测结果仅展示置信度和概率柱状图，无法可视化"音频中哪一段触发了预测"。
- **无测验或评估模块**：平台目前缺乏检验学生理解程度或记录学习成果的机制。

## 未来工作方向

### 1. 扩展与多样化数据集

- 引入多语言数据集（普通话、西班牙语、法语等），提升跨语言、跨口音的泛化能力。
- 持续更新数据集，纳入新发布 AI 语音系统（如 ElevenLabs、Suno、Kling Audio 等）的合成样本。
- 添加含噪声、压缩失真的音频变体，以评估模型在真实环境中的鲁棒性。

### 2. 更丰富的音频表示

- 将均值聚合特征替换为**序列级表示**（如逐帧 MFCC 序列，输入 RNN 或 Transformer 模型）。
- 引入**韵律特征**：基频（F0）轮廓、语速、能量动态变化。
- 探索**原始波形模型**（如 RawNet2、Wav2Vec 2.0 微调版），直接从音频波形端到端学习，无需手工特征。

### 3. 模型能力提升

- 添加**注意力可视化**：高亮显示对预测贡献最大的时间段，使模型决策对学生透明可解释。
- 支持**多分类输出**：在判断真假的基础上，进一步识别可能使用的生成方式（TTS、变声、拼接合成等）。
- 引入**集成预测**：将 SpecRNet 与多个 ML 模型的输出通过加权投票融合，提升整体准确率。
- 评估**对抗鲁棒性**：测试模型面对专门设计用于欺骗检测器的对抗性音频时的表现。

### 4. 实时检测

- 基于 WebSocket 或 WebRTC 实现**流式推理**，支持对直播或实时通话中的音频进行近实时检测。
- 重新引入**浏览器端录音与即时分析**功能（配合完善的 UX 设计），对音频分块处理并实时返回结果。

### 5. 平台与基础设施

- 添加**用户账户与学习进度追踪**，支持教师查看每位学生已运行和查阅的模型情况。
- 实现**按用户隔离的结果缓存**，避免并发会话相互干扰。
- 通过 NVIDIA Container Toolkit 支持 **GPU 推理**，加速 SpecRNet 评估。
- 引入 **Let's Encrypt 自动续签证书**，无需依赖 Cloudflare 即可实现生产级 HTTPS。
- 提供**一键云部署模板**（如 Railway、Render 或 Hugging Face Spaces），方便没有本地服务器的学校快速上线。

### 6. 教学功能增强

- **引导式教程模式**：设计逐步引导流程，带领学生按"数据 → 特征 → 训练 → 评估"的顺序学习。
- **随堂测验模块**：在每个章节后嵌入理解性问题（如"哪个模型的 AUC 最高？SVM 为何在此场景优于 KNN？"）。
- **反事实探索器**：允许学生手动调整特征值，观察预测结果的变化，直观建立对特征重要性的理解。
- **作业导出功能**：支持教师将模型对比表、混淆矩阵等以 PDF 报告形式导出，用于学生作业提交。

### 7. 多语言与无障碍支持

- 在中英文之外，将国际化（i18n）扩展至日语、韩语、西班牙语等，满足国际化教学需求。
- 添加 ARIA 标签和键盘导航支持，符合无障碍访问规范。
- 为图表提供**屏幕阅读器友好的文字描述**，辅助视觉障碍用户使用。

## 总结对比

| 维度 | 当前状态 | 未来方向 |
|------|---------|---------|
| 数据集 | 11,778 条，8 位说话人，仅英语 | 多语言，持续更新 |
| 特征 | 26 维均值统计量 | 时序序列、韵律特征、原始波形 |
| 模型 | 6 个 ML + 1 个 DL（二分类） | 集成学习、多分类、对抗鲁棒性 |
| 推理方式 | 文件上传，CPU 推理 | 实时流式，GPU 加速 |
| 可解释性 | 仅置信度分数 | 注意力热图、反事实探索 |
| 部署 | 单节点 Docker，自签名证书 | 多用户隔离，Let's Encrypt，云模板 |
| 教学功能 | 自由探索 | 引导教程、随堂测验、作业导出 |
